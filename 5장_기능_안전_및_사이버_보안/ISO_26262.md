# ISO 26262 (Functional Safety)

## 개요
ISO 26262는 자동차의 전기/전자(E/E) 시스템의 오류로 인한 사고를 방지하기 위해 제정된 기능 안전(Functional Safety) 국제 표준입니다. 의료 로봇 분야에서는 IEC 60601-1 (의료 기기 안전) 표준이 주로 사용되지만, 자율주행 기술과 고도의 전자 제어 시스템이 도입되면서 ISO 26262의 개념과 프로세스가 점점 더 중요하게 다루어지고 있습니다.

### 핵심 개념
기능 안전이란, 시스템이 의도치 않게 작동하거나 고장 났을 때 발생할 수 있는 위험을 허용 가능한 수준으로 낮추는 것을 말합니다.

1.  **ASIL (Automotive Safety Integrity Level)**: 위험도에 따라 A, B, C, D 4단계로 등급을 분류합니다. ASIL D가 가장 엄격한 안전 요구사항을 가집니다. (의료 기기의 Class I, II, III와 유사)
2.  **Safety Lifecycle**: 제품의 기획부터 폐기까지 전 과정(Concept -> Design -> Development -> Validation -> Production -> Decommissioning)에 걸쳐 안전 활동을 정의합니다.
3.  **HARA (Hazard Analysis and Risk Assessment)**: 발생 가능한 위험원을 식별하고, 심각도(Severity), 노출 빈도(Exposure), 제어 가능성(Controllability)을 평가하여 ASIL 등급을 결정합니다.

### Ethernet 통신과 기능 안전
Ethernet을 사용하는 의료 로봇 시스템에서 기능 안전을 확보하기 위해 다음과 같은 기술들이 적용됩니다.
-   **Black Channel Communication**: 통신망(Ethernet 스위치, 케이블 등)을 신뢰할 수 없는 'Black Channel'로 간주하고, 데이터의 무결성을 보장하기 위해 상위 계층(Application Layer)에서 별도의 안전 프로토콜(Safety Layer)을 추가합니다. (예: CRC, Sequence Number, Timeout Monitoring)
-   **E2E (End-to-End) Protection**: 송신 ECU에서 생성된 데이터가 수신 ECU까지 변조되거나 손실되지 않고 안전하게 전달되었는지 확인하는 메커니즘입니다. AUTOSAR에서는 E2E Library를 제공하여 이를 지원합니다.
-   **Freedom from Interference (FFI)**: 서로 다른 ASIL 등급을 가진 소프트웨어 모듈이 하나의 제어기에서 실행될 때, 낮은 등급의 모듈 오류가 높은 등급의 모듈에 영향을 주지 않도록 메모리 보호(MPU), 시간 파티셔닝(Time Partitioning) 등을 적용해야 합니다.

### 의료 로봇 분야 적용
수술 로봇의 경우, 로봇 팔의 오작동은 환자에게 치명적인 상해를 입힐 수 있으므로 최고 수준의 안전 등급이 요구됩니다. ISO 26262의 프로세스를 따름으로써, 하드웨어 고장(Random Hardware Failure)뿐만 아니라 소프트웨어 버그(Systematic Failure)까지 체계적으로 관리하고 예방할 수 있습니다. 특히 이중화(Redundancy) 설계와 결합하여, 하나의 센서나 제어기가 고장 나더라도 안전하게 정지하거나 기능을 유지(Fail-Operational)할 수 있는 아키텍처를 구현해야 합니다.
