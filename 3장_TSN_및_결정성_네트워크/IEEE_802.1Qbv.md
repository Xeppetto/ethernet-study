# IEEE 802.1Qbv (TAS: Time Aware Shaper)

## 개요
Ethernet 네트워크에서 트래픽의 종류와 중요도는 다양합니다. 특히 의료 로봇과 같이 실시간 제어가 필요한 시스템에서는 제어 명령(Control Traffic)과 같은 중요 데이터를 다른 트래픽(예: 로그 파일 전송)보다 우선적으로 처리해야 합니다. 하지만 일반적인 이더넷 스위치는 'Best Effort' 방식으로 동작하여, 트래픽이 몰리면 지연(Latency)이 발생할 수 있습니다. IEEE 802.1Qbv는 이러한 문제를 해결하기 위해 도입된 '시간 지정 스케줄러(Time Aware Shaper, TAS)' 기술입니다.

### 동작 원리
TAS는 스위치의 각 출력 포트에 '게이트(Gate)' 개념을 도입하여 트래픽 종류별로 전송 시간을 할당합니다. 마치 기차역의 타임테이블처럼, 특정 시간에는 특정 종류의 트래픽만 통과할 수 있도록 제어합니다.

1.  **GCL (Gate Control List)**: 스위치 내부에 트래픽 전송 일정을 정의한 테이블입니다.
2.  **Cycle Time**: 전체 스케줄 주기를 의미하며, 이 주기 내에서 각 게이트의 열림(Open)과 닫힘(Closed) 상태가 반복됩니다.
3.  **Scheduled Traffic (ST)**: 제어 명령과 같이 실시간성이 보장되어야 하는 트래픽입니다. GCL에 의해 정해진 시간에만 전송됩니다.
4.  **Best Effort (BE)**: 일반 데이터 트래픽입니다. ST가 전송되지 않는 시간에만 전송될 수 있습니다.

### 주요 기능
- **시간 분할**: 1ms 주기로 동작한다고 가정할 때, 앞의 100µs는 제어 데이터만 전송하고, 나머지 900µs는 일반 데이터를 전송하도록 설정할 수 있습니다.
- **가드 밴드 (Guard Band)**: BE 트래픽이 전송 중에 ST 전송 시간이 도래하면 충돌이 발생할 수 있습니다. 이를 방지하기 위해 ST 전송 직전에 미리 BE 전송을 차단하는 보호 구간을 둡니다.

### 의료 로봇에서의 활용
수술 로봇의 경우, 의사의 손동작을 로봇 팔에 전달하는 명령(ST)은 지연 없이 즉시 전송되어야 합니다. 반면 수술 영상을 녹화하여 서버로 보내는 데이터(BE)는 다소 지연되더라도 큰 문제가 없습니다. 802.1Qbv를 적용하면, 영상 데이터 전송 중이라도 제어 명령이 도착하면 즉시(또는 정해진 시간에 정확히) 전송되도록 보장할 수 있습니다. 이를 통해 **확정적인 지연 시간(Deterministic Latency)**과 **극도로 낮은 지터(Low Jitter)**를 달성하여 로봇의 정밀 제어 성능을 극대화합니다.
