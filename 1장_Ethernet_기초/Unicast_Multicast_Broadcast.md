# Unicast, Multicast, Broadcast

## 개요
네트워크에서 데이터를 전송하는 방식은 크게 세 가지로 나눌 수 있습니다: Unicast(유니캐스트), Multicast(멀티캐스트), Broadcast(브로드캐스트). 이들은 수신 대상의 범위와 전송 방법에 따라 구분되며, Ethernet 네트워크의 효율성과 성능에 직접적인 영향을 미칩니다. 의료 로봇과 같은 실시간 시스템에서는 각 통신 방식의 특성을 이해하고 적절히 활용하는 것이 중요합니다.

### Unicast (1:1 통신)
Unicast는 가장 일반적인 통신 방식으로, 송신자 한 명이 수신자 한 명에게 데이터를 보내는 방식입니다. 마치 전화 통화처럼 특정 대상과 1:1로 통신하는 것과 같습니다. 수신 측의 MAC 주소와 IP 주소를 명확히 알고 있어야 하며, 다른 장비들은 해당 패킷을 무시합니다.

#### 특징 및 활용
- **효율성**: 필요한 대상에게만 데이터를 전송하므로 네트워크 대역폭을 낭비하지 않습니다.
- **제어 메시지**: 로봇의 특정 관절 제어 명령이나 센서 데이터 요청과 같이 1:1 통신이 필요한 경우 주로 사용됩니다.
- **신뢰성**: TCP와 같은 연결 지향 프로토콜과 결합하여 데이터의 정확한 전달을 보장할 수 있습니다.

### Multicast (1:N 통신)
Multicast는 송신자 한 명이 특정 그룹에 속한 여러 수신자에게 동시에 데이터를 보내는 방식입니다. 그룹에 가입된 장비들만 데이터를 수신하며, 네트워크 장비(스위치, 라우터)의 지원이 필요합니다.

#### 특징 및 활용
- **대역폭 절약**: 같은 데이터를 여러 곳에 보낼 때 Unicast를 여러 번 보내는 것보다 효율적입니다.
- **실시간 스트리밍**: 영상 전송, 음성 회의 등 다수의 사용자에게 동일한 데이터를 실시간으로 전송할 때 유리합니다.
- **로봇 시스템**: 로봇 내의 여러 센서 데이터를 다수의 제어기(ECU)가 동시에 공유해야 하는 경우(예: Publish/Subscribe 모델의 DDS 통신) 효과적입니다.
- **IGMP Snooping**: 스위치에서 멀티캐스트 트래픽을 효율적으로 관리하기 위해 IGMP Snooping 기술이 사용되기도 합니다.

### Broadcast (1:All 통신)
Broadcast는 송신자 한 명이 네트워크에 연결된 모든 장비에게 데이터를 보내는 방식입니다. 수신자의 주소를 특정하지 않고, 모든 장비가 해당 패킷을 수신하고 처리해야 합니다.

#### 특징 및 활용
- **네트워크 발견**: 새로운 장비가 네트워크에 접속하여 IP 주소를 할당받거나(DHCP), 상대방의 MAC 주소를 찾을 때(ARP) 주로 사용됩니다.
- **부하 발생**: 불필요한 트래픽을 유발하여 네트워크 전체 성능을 저하시킬 수 있으므로(Broadcast Storm), 신중하게 사용해야 합니다.
- **제한적 사용**: 최근에는 불필요한 브로드캐스트를 줄이고, 멀티캐스트나 유니캐스트로 대체하려는 추세입니다.

### 의료 로봇 네트워크 설계 시 고려사항
의료 로봇의 내부 네트워크는 안정성과 실시간성이 매우 중요합니다. 무분별한 Broadcast 사용은 네트워크 혼잡을 초래하여 제어 명령의 지연(Latency)이나 손실(Packet Loss)을 유발할 수 있습니다. 따라서 제어 데이터는 Unicast나 Multicast를 적절히 활용하여 전송하고, Broadcast는 ARP와 같은 필수적인 경우에만 제한적으로 사용하는 것이 바람직합니다. 특히 TSN(Time Sensitive Networking) 환경에서는 트래픽의 종류에 따라 우선순위를 부여하고 대역폭을 할당하므로, 통신 방식의 선택은 더욱 중요해집니다.
