# MAC 주소와 IP 주소의 차이 (MAC vs IP Address)

## 개요
네트워크 통신에서 MAC 주소와 IP 주소는 모두 장치를 식별하기 위한 고유한 값입니다. 하지만 그 역할과 사용되는 계층, 범위가 다르며, 이 두 가지 주소의 차이를 명확히 이해하는 것은 Ethernet 기반 의료 로봇을 개발하고 디버깅하는 데 있어 필수적입니다. 물리적 주소와 논리적 주소의 개념을 이해하고, 서로 어떻게 상호 작용하는지 알아보겠습니다.

### MAC 주소 (Media Access Control Address)
MAC 주소는 네트워크 인터페이스 카드(NIC)에 할당된 고유 식별자입니다. 물리 계층(L1)과 데이터 링크 계층(L2) 사이에서 통신을 가능하게 하는 주소로, 제조사에서 하드웨어적으로 할당하여 변경이 불가능합니다. 48비트(6바이트)의 16진수로 표현되며, 전 세계적으로 유일한 값을 가집니다.

#### 특징
- **물리적 주소**: 하드웨어 자체에 부여된 고유한 식별자입니다.
- **범위**: 로컬 네트워크 내(동일 서브넷)에서 장치를 식별하는 데 사용됩니다.
- **불변성**: 일반적으로 제조 시 부여되어 변경되지 않습니다. (소프트웨어적으로 변경 가능하나 권장되지 않음)

### IP 주소 (Internet Protocol Address)
IP 주소는 네트워크 계층(L3)에서 장치를 식별하고 통신하기 위한 논리적 주소입니다. 인터넷 프로토콜(IP)을 사용하는 네트워크에서 송수신 장치를 구별하고, 데이터 패킷이 목적지까지 도달할 수 있는 경로를 찾는 데 사용됩니다. IPv4는 32비트, IPv6는 128비트로 구성됩니다.

#### 특징
- **논리적 주소**: 네트워크 관리자에 의해 할당되거나 DHCP를 통해 동적으로 변경될 수 있습니다.
- **범위**: 전 세계 네트워크(인터넷) 상에서 유일한 식별자로 사용됩니다.
- **계층적 구조**: 네트워크 주소와 호스트 주소로 나뉘어 있어 라우팅에 유리합니다.

### 의료 로봇에서의 활용
의료 로봇 시스템 내에서 여러 제어기(ECU)가 Ethernet 스위치를 통해 연결될 때, 각 ECU는 고유한 MAC 주소를 가집니다. 스위치는 이 MAC 주소를 기반으로 프레임을 전달합니다. 반면, 로봇이 외부 병원망이나 클라우드 서버와 통신할 때는 IP 주소가 필요합니다. 라우터나 게이트웨이는 IP 주소를 보고 패킷을 전달하며, 내부망에서는 ARP(Address Resolution Protocol)를 통해 IP 주소에 해당하는 MAC 주소를 알아내어 통신을 수행합니다. 따라서 통신 장애 시, MAC 주소 테이블이나 ARP 캐시를 확인하는 것이 문제 해결의 첫걸음이 될 수 있습니다.
